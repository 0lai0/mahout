## Problem

`new_synthetic`, `new_from_file`, and `new_from_file_streaming` duplicate ~200 lines of normalize-config → build-producer → spawn-thread logic in `pipeline_runner.rs`.

## Scope

```rust
pub enum Source {
    Synthetic { total_batches: usize, seed: Option<u64> },
    File(PathBuf),
    FileStreaming(PathBuf),
}

impl PipelineIterator {
    pub fn new(engine: QdpEngine, source: Source, config: PipelineConfig) -> Result<Self> { ... }
}
```

- Keep old `new_*` as thin wrappers **or** update PyO3 call sites in the same PR
- **No producer logic change** in this issue ([E2](E2.md) handles producers)
