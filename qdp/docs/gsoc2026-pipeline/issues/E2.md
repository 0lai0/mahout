## Problem

`SyntheticProducer`, `InMemoryProducer`, and `StreamingProducer` each copy the same recycle-or-allocate pattern (~80% identical `produce()` bodies).

## Scope

- `fn reuse_or_alloc<T: FloatElem>(recycled: Option<Vec<T>>, len: usize) -> Vec<T>`
- Minimal adapters: `SyntheticAdapter`, `InMemoryAdapter`, `StreamingAdapter`
- `AdapterBackedProducer` implements `BatchProducer`, replacing the three structs
- Bit-identical output vs legacy producers (test)

**Out of scope:** VecDeque ([E3](E3.md)) unless same assignee combines PRs
