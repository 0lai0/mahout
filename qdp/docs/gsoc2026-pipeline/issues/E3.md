## Problem

Streaming path advances its buffer with `Vec::drain(..cursor)` — O(n) memmove per batch. Use `VecDeque` so the hot path is O(take) not O(buffer.len()).

## Scope

- Replace `Vec` + `drain` with `VecDeque` in streaming adapter / producer
- Preserve sample-boundary and chunk-read behavior

## Files expected

- `qdp-core/src/pipeline_runner.rs`
